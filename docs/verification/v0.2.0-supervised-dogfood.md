# v0.2.0 Supervised Dogfood Verification

Use this playbook after a v0.2.0 release to run one deliberately small dogfood ticket through the supervised pipeline.

The v0.2.0 flow includes independent agent review, bounded remediation, smoke verification, validation, PR creation, CI waiting, and Linear handoff.

## Operator Steps

1. Pick a tiny documentation-only ticket that is already ready for development.
2. Confirm the ticket has a clear scope, acceptance criteria, and no requested product or release artifact changes.
3. Start the supervised run against that single ticket and keep the run bounded to the requested file area.
4. Review the independent agent findings before allowing any remediation work.
5. Allow only narrow remediation that directly addresses the review or validation result.
6. Run smoke verification and the repo validation commands required by the ticket.
7. Create the PR only after the working tree contains the expected tiny diff and validation has passed.
8. Wait for CI to finish, then complete the Linear handoff with the PR and validation summary.

## Safe Ticket Checklist

- The diff is documentation-only.
- Exactly one operator-owned ticket is in scope.
- The run has a clear stop point if review, validation, or CI fails.
- The handoff summary lists the changed file, validation results, and any residual risk.
